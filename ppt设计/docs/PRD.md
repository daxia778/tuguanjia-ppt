## 1. 产品概述

### 1.1 产品定位
SlideForge 是一个面向**团队内部使用**的 AI PPT 自动生成工具。用户通过自然语言描述需求，AI 自动完成内容拆解、页面设计、视觉生成，最终输出可下载的 `.pptx` 文件。

### 1.2 核心价值
| 维度 | 传统方式 | SlideForge |
|------|---------|------------|
| 制作时间 | 2-8小时 | 2-5分钟 |
| 设计能力 | 需专业设计功底 | AI自动设计 |
| 成本 | 设计师费用高 | ¥0.3/份 |
| 一致性 | 风格差异大 | 统一视觉标准 |

### 1.3 目标用户
- 中国大陆团队内部成员
- 需要快速制作演示文稿的业务人员
- 无专业设计经验但有高质量输出需求的用户

### 1.4 使用场景
- 团队周报/月报快速生成
- 项目提案、商业计划书
- 产品发布会、技术分享
- 市场分析报告、数据报告

## 2. 功能需求

### 2.1 P0 — 核心功能 (MVP)

#### 2.1.1 自然语言输入
- **描述**: 用户在文本框中输入PPT主题、风格、页数等需求
- **交互**: 支持多行文本输入，实时字数统计
- **示例**: *"帮我做一份10页的新能源汽车市场分析报告，科技感风格，深色主题"*

#### 2.1.2 风格预设选择
- **预设风格**: 科技感、极简、商务、创意、自然、暗黑（共6种）
- **视觉预览**: 每种风格有渐变色缩略图预览
- **默认选择**: 科技感（tech）

#### 2.1.3 页数设定
- **范围**: 3-20页
- **默认值**: 8页
- **交互**: 加减按钮控制

#### 2.1.4 AI 智能拆解
- **能力**: 将用户描述自动拆分为多页结构
- **页面类型**: 封面页、目录页、内容页、数据页、结尾页
- **输出**: 每页的标题、内容大纲、布局建议

#### 2.1.5 逐页生图 (GPT Image-2)
- **模型**: OpenAI GPT Image-2 (通过中转API调用)
- **分辨率**: 2K (2048px)
- **格式**: 每页生成一张完整的PPT设计图
- **状态追踪**: pending → generating → done 三态

#### 2.1.6 实时预览
- **主预览区**: 大尺寸预览当前选中页面
- **胶片条**: 侧边栏所有页面缩略图列表
- **页面导航**: 上一页/下一页切换，支持点击缩略图跳转
- **状态展示**: 生成中（旋转加载）、等待中（时钟图标）、已完成（勾选标记）

#### 2.1.7 单页重新生成
- **功能**: 对不满意的页面可单独重新生成
- **交互**: 点击"重新生成此页"按钮
- **动画**: 卡片缩放闪烁反馈

#### 2.1.8 导出 PPTX
- **格式**: 标准 .pptx 文件
- **内容**: 将所有AI生成的图片嵌入PPT页面
- **下载**: 一键打包下载

### 2.2 P1 — 增强功能 (后续迭代)

|------|------|------|
| Prompt 编辑 | 对单页的生成提示词进行微调 | 规划中 |

## 3. 技术架构

### 3.1 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19.2.5 | UI框架 |
| Vite | 8.0.10 | 构建工具 |
| React Router | 7.14.2 | 路由管理 |
| GSAP | 3.15.0 | 高级动画引擎 |
| @gsap/react | 2.1.2 | React GSAP 集成 |
| Lenis | 1.3.23 | 平滑滚动 |
| Lucide React | 1.11.0 | 图标库 |

### 3.2 后端架构 (待实现)

```
┌─────────────┐     ┌──────────────┐     ┌────────────────┐
│  前端 React  │────▶│  API Gateway  │────▶│ 中转站 API      │
│  (Vite)      │◀────│  (Go / Node)  │◀────│ (GPT Image-2)  │
└─────────────┘     └──────────────┘     └────────────────┘
                            │
                     ┌──────▼──────┐
                     │   文件存储    │
                     │ (本地 / OSS) │
                     └─────────────┘
```

- **API Gateway**: Go 语言 (`test_api.go`)，负责请求转发、鉴权、速率限制
- **AI 模型**: 通过第三方中转站调用 OpenAI GPT Image-2
- **文件处理**: 生成的图片打包封装为 `.pptx` 文件

### 3.3 部署方案
- **运行环境**: 中国大陆内网
- **前端**: Vite 开发服务器 / Nginx 静态部署
- **开发端口**: `8111`
- **后端**: Docker 容器化部署
- **域名**: 内网域名或 IP 直连

## 4. 用户流程

```mermaid
graph TD
    A[打开 SlideForge] --> B[浏览首页]
    B --> C{点击"开始创作"}
    C --> D[输入主题描述]
    D --> E[选择风格 + 页数]
    E --> F[点击"开始生成"]
    F --> G[AI 拆解页面结构]
    G --> H[逐页调用 GPT Image-2 生图]
    H --> I[实时预览生成结果]
    I --> J{满意?}

    K --> I
    J -- 满意 --> L[一键导出 PPTX]
    L --> M[下载文件]
```

## 5. 页面规划

### 5.1 首页 (Landing Page) — `/`
- **导航栏**: 品牌Logo、案例/功能/流程锚点、开始创作CTA
- **Hero 区**: 核心标题动画、产品数据统计、CTA按钮组
- **案例展示**: 5张卡片滚动翻页动画（GSAP ScrollTrigger驱动）
- **功能介绍**: 4张功能卡片（自然语言驱动、逐页生图、实时预览、一键导出）
- **工作流程**: 3步流程时间线（描述→AI生成→预览导出）
- **底部CTA**: 全幅行动号召区域
- **页脚**: 品牌标识

### 5.2 创作页 (Create Page) — `/create`
- **输入阶段**: Prompt输入框、风格选择、页数设定
- **预览阶段**: 大尺寸预览 + 侧边胶片条 + 操作按钮
- **状态转换**: input → preview 过渡动画

## 6. 设计规范

### 6.1 色彩体系
- **主色**: `#6366f1` (Indigo-500)
- **强调色**: `#ec4899` (Pink-500)
- **背景**: `#07080a` → `#0e0e1a` (深色渐变)
- **文字**: `#ffffff` / `rgba(255,255,255,0.7)` / `rgba(255,255,255,0.4)`
- **渐变标题**: `linear-gradient(135deg, #f97316, #ec4899, #8b5cf6)`

### 6.2 字体
- **显示字体**: Inter (Google Fonts)
- **中文回退**: system-ui, -apple-system

### 6.3 间距系统
- **小**: 8px
- **中**: 16px / 24px
- **大**: 32px / 48px
- **超大**: 64px / 80px

### 6.4 圆角
- **小**: 8px
- **中**: 12px / 16px
- **大**: 20px / 24px
- **全圆**: 999px

### 6.5 动效规范
- **入场动画**: power3.out 缓动
- **滚动驱动**: GSAP ScrollTrigger scrub
- **平滑滚动**: Lenis (duration: 1.2)
- **微交互**: hover scale/shadow 变化

## 9. 风险与约束

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 中转API稳定性 | 生图失败 | 重试机制 + 多通道备份 |

| 生成质量波动 | 用户不满 | 单页重新生成 + Prompt优化 |
