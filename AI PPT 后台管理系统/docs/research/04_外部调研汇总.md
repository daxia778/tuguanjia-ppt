## 一、支付通道资质门槛

### 1.1 微信支付（个体工商户）

**结论：能申请，功能与企业一致**

| 项目 | 详情 |
|------|------|
| 入口 | pay.weixin.qq.com → 注册商户号 → 个体工商户 |
| 审核周期 | 1-3 个工作日（短信/邮件通知） |
| 材料 | 营业执照 + 经营者身份证 + 个人银行卡 |
| 费率 | 标准 0.6%（小微/个体享 9 折优惠） |
| 结算 | 可绑定经营者**个人银行卡**（无需对公账户） |
| 功能 | Native/JSAPI/退款/对账全支持 |
| 限制 | 高风险类目审核更严，单笔/日限额可能略低 |

**信息来源**：微信支付官方《提交申请单_商户进件》（2025.02.28 更新）

### 1.2 支付宝

**结论：个体户可接入当面付**

| 项目 | 详情 |
|------|------|
| 入口 | b.alipay.com → 支付产品 → 当面付 |
| 审核周期 | 1-3 个工作日 |
| 提供执照 | 解除额度限制 |
| 结算 | 无需强制对公账户 |

**信息来源**：支付宝官方《当面付产品开通申请条件》（2025.11 更新）

### 1.3 第三方聚合支付（备选方案）

| 平台 | 费率 | 门槛 | 风险 |
|------|------|------|------|
| 虎皮椒（xunhupay） | ~0.6% + 服务费 | 个人实名 | 风控冻结 |
| PAYJS | 开户费300 + 1.78% | 个人实名 | T+1，高频冻结 |

**结论**：正式项目建议走个体户 + 官方通道，聚合支付仅作过渡。

### 1.4 个体工商户注册

| 项目 | 详情 |
|------|------|

## 三、.pptx 文件在线预览

### 方案对比

| 方案 | 类型 | 成本 | 兼容性 | 推荐度 |
|------|------|------|--------|--------|
| **pptx-preview** | 前端库 | 免费 | 基本（文本/图片/图表） | ⭐⭐⭐⭐ Admin端首选 |
| LibreOffice headless | 服务端 | 免费 | 好（转PDF/图片） | ⭐⭐⭐ |
| WPS WebOffice SDK | 云服务 | 按量付费 | 最好（40+格式） | ⭐⭐⭐⭐⭐ 预算允许首选 |
| Aspose.Slides | 商业 | 贵 | 最好 | ⭐⭐ 成本高 |

### pptx-preview 详情

- **GitHub**：[501351981/pptx-preview](https://github.com/501351981/pptx-preview)
- **npm 包**：`pptx-preview`
- **支持**：文本、图片、图表、基本形状
- **不支持**：复杂动画/过渡/3D
- **集成难度**：Vue3 直接引入，10 行代码搞定

### 模板填充方案

| 工具 | 语言 | 能力 | 性能 |
|------|------|------|------|
| **python-pptx** | Python | 完整填充（文本/图片/图表/表格/母版） | 20页 1-5秒 |
| pptxgenjs | Node.js | 生成为主 | 快 |
| node-pptx | Node.js | 有限编辑 | 一般 |

**推荐**：AI 填充模板走 Python 服务（FastAPI），后端主体 Java。

| 框架 | 生态活跃度 | RBAC | 中文文档 | 推荐度 |
|------|----------|------|---------|--------|
| **ruoyi-vue-pro** | ⭐⭐⭐⭐⭐ | 完整+支付 | 完善 | **首选** |
| RuoYi-Vue3 | ⭐⭐⭐⭐ | 完整 | 完善 | 备选 |

| Arco Design Pro | ⭐⭐⭐ | 完整 | 好 | 字节生态 |

**结论**：ruoyi-vue-pro 已内置支付+积分+订单，直接 fork 最省时。

### RuoYi 代码生成器

自动生成：Entity、Mapper、Service、Controller、SQL + Vue3 页面、API、路由、菜单  
**不支持**：完整 RBAC 按钮权限（需手动补充）

### 常见坑

- 权限配置：菜单同步后需重启/清缓存
- Vue3 版本不一致导致组件报错
- 代码生成器模板可能需适配最新 Element Plus

### 开源项目

| 项目 | 积分模块位置 | 特点 |
|------|-------------|------|
| CRMEB (Java) | MemberPointsServiceImpl.java | 含退款回滚逻辑 |

| youlai-mall | 会员积分 | Spring Cloud 微服务 |
| ruoyi-vue-pro | 会员积分模块 | 已集成 |

### 退款积分扣回业界做法

- 主流平台（拼多多/美团）：异步回调中生成**负向流水**，扣减已发放积分
- CRMEB：退款金额为 0 时仍需回滚积分

### 支付回调幂等标准做法

```
唯一订单号 + 数据库/Redis 锁 + "已处理"标记 + 签名验证 + 金额校验
```

## 七、ICP 备案

| 场景 | 需要备案？ | 说明 |
|------|----------|------|
| 内网/局域网部署 | ❌ 不需要 | 未公网域名解析 |
| 公网部署（个人） | 🟡 能但受限 | 不能有经营性内容/支付 |
| 公网部署（个体户） | ✅ 推荐 | 3-15 天，支持经营性 |

### 企业 ICP 备案

- 周期：3-15 个工作日（材料齐全+核验通过）
- 材料：营业执照 + 法人身份证 + 网站负责人 + 域名 + 服务器
- 部分省市支持"秒批"

## 九、额外发现

1. **ruoyi-vue-pro vs 基础 RuoYi**：前者多了支付/积分/商城/AI 模块，后者是纯后台管理。项目应 fork 前者。
2. **python-pptx 才是模板填充最佳方案**：如果后续要做"AI 填充模板占位符"，建议搭一个 Python 微服务（FastAPI）。
3. **EasyExcel 流式导出**：100w+ 行不 OOM，是 Java 生态 Excel 导出的标准方案。
4. **拖拽排序**：推荐 debounce 200ms 后批量提交，不要每次拖拽都发请求。
