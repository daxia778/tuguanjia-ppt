# 代码实现 vs 外部审批 — 全景拆分

> 核心思路：你能控制的（写代码）和你控制不了的（等审批）要并行推进  
> 关键结论：**没有任何外部审核会阻塞开发**

---

## 二、外部审核报备（你控制不了的部分）

### 时间线（建议 Day 1 全部启动）

```
Day 1 ──────────────────────────────────────────────→

├─ 🔴 微信支付商户号申请 ────── 审核 1-7天 ────→ ✅ 拿到商户号
│    入口：pay.weixin.qq.com → 注册 → 个体工商户
│    材料：营业执照 + 身份证 + 个人银行卡
│    操作人：老板（微信扫码确认）
│    ⚠️ 类目选"软件/信息技术服务"

├─ 🟡 支付宝当面付签约 ────── 审核 1-3天 ────→ ✅ 拿到 PID/密钥
│    入口：b.alipay.com → 支付产品 → 当面付
│    操作人：老板

├─ 🟡 阿里云企业认证 ────── 1天 ────→ ✅ 解锁短信/OSS
│    上传营业执照 + 法人身份证
│    操作人：你

└─ 🟢 ICP备案（仅公网需要）── 3-15天 ──→ ✅ 域名可公网
     内网部署 = 不需要
```

### 外部依赖清单

| 事项 | 谁操作 | 材料 | 周期 | 阻塞开发？ |
|------|--------|------|------|-----------|
| 微信支付商户号 | 老板扫码 | 执照+身份证+银行卡 | 1-7天 | ❌ 沙箱可先开发 |
| 支付宝签约 | 老板登录 | 同上 | 1-3天 | ❌ |
| 云服务企业认证 | 你 | 执照扫描件 | 1天 | ❌ |
| ICP备案 | 你 | 执照+域名+服务器 | 3-15天 | ❌ 内网不需要 |
| 短信签名审核 | 提交后等 | 企业认证后 | 1-3天 | ❌ 可用邮箱替代 |

---

### 政策规则限制（红线）

| 规则 | 影响 | 应对 |
|------|------|------|
| 个体户费率 0.6% | 每笔交易手续费 | 定价时计入成本 |
| T+1 结算到个人银行卡 | 非实时到账 | 财务模块标注 |
| 退款 1-3 天到账 | 非实时 | 前端提示用户 |
| 大额交易风控 | 单日大额可能暂停 | 初期小额测试 |
| 回调必须返回 SUCCESS | 否则重试 24h | 代码兜底 |
| 密钥泄露风险 | 被盗刷 | 密钥不进 Git |
