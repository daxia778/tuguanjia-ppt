## 一、整体判断

> [!IMPORTANT]
> **实现难度：中等** — 有成熟开源框架（ruoyi-vue-pro）兜底，真正的技术难点只有积分-订单-退款联动。
> 所有外部审批（支付、备案等）均不阻塞开发。

**关键发现**：ruoyi-vue-pro（YunaiV 维护）已内置完整的支付宝 + 微信支付 + 退款 + 订单 + RBAC + 操作日志模块，直接 fork 二开可省 10-15 个工作日。

## 三、三层难度分布

### 🟢 第一层：开箱即用（4.5天，13%）
ruoyi-vue-pro 已有，改配置就行：RBAC 权限、操作日志、系统配置、支付核心。

### 🟡 第二层：代码生成器 + 手写（16天，48%）
用 RuoYi 代码生成器出 CRUD，然后手写业务逻辑：用户管理、模板管理、风格库、通知、邀请等。

### 🔴 第三层：需精细设计（13天，39%）
需要认真设计的核心模块：积分系统、仪表盘、财务统计、内容安全、AI配置管理。

## 五、技术选型决策

| 层次 | 选型 | 理由 |
|------|------|------|
| **框架** | ruoyi-vue-pro (fork) | 自带支付+积分+权限+日志，省 10-15 天 |
| **前端** | Vue3 + Element Plus | 国内生态最强，中文文档完善 |
| **后端** | Spring Boot 2.7+ | ruoyi-vue-pro 默认，生态成熟 |
| **数据库** | MySQL 8.0 + Redis | 默认配置，无需折腾 |
| **图表** | ECharts 5 | 国产、文档好 |
| **富文本** | wangEditor 5 | 国产开源，中文支持最佳 |
| **PPTX预览** | pptx-preview（前端库） | 纯前端，免部署 LibreOffice |
| **敏感词** | ToolGood.Words + DFA | 开源 14w+ 词库，毫秒级 |
| **Excel导出** | EasyExcel（阿里） | 流式写入，100w+ 行不 OOM |
| **文件存储** | 阿里云 OSS | 前端直传（STS 临时凭证） |
| **短信替代** | 邮箱验证码 + TOTP | 腾讯云 2025.9 已停止个人短信新增 |
| **支付SDK** | weixin-java-pay + alipay-sdk-java | ruoyi-vue-pro 已集成 |

## 六、风险评估

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| 积分-退款数据不一致 | 财务错误 | 中 | 乐观锁 + 事务 + 流水对账 |
| 微信支付风控冻结 | 无法收款 | 低 | 初期小额测试，不选高风险类目 |
| pptx-preview 渲染偏差 | 预览不准确 | 中 | Admin 端预览够用，不影响生成 |
| 大数据量仪表盘卡顿 | 体验差 | 低 | 定时任务预计算统计表 |
| 支付回调丢失 | 用户付款但积分未到 | 低 | 定时补偿查询 + 手动补发 |
